# 🔘 Инструкция по Умным Кнопкам-Скриптам

Умные кнопки (Smart Buttons) — это мощный визуальный инструмент в Obsidian Maker. Они позволяют превращать обычный текст в интерактивные кнопки, по клику на которые выполняются полезные действия: создание заметок, вызов AI, запуск внутренних команд Obsidian и многое другое.

---

## 🏗️ Как создать кнопку

Чтобы создать кнопку, добавьте в любую вашу заметку блок кода с языком `om-button`. Внутри укажите настройки в формате YAML (ключ: значение).

**Базовый пример:**
```yaml
```om-button
label: Моя первая кнопка
action: create-note
filename: Новая заметка
color: primary
```
```

---

## 🎨 Настройка внешнего вида

Вы можете сделать кнопку уникальной, используя параметры цвета, стиля и иконок.

- **`label`** (обязательно): Текст, который будет написан на кнопке.
- **`icon`**: Название иконки. Плагин поддерживает все иконки из библиотеки [Lucide](https://lucide.dev/icons) (например: `check-square`, `file-plus`, `robot`, `calendar`).
- **`color`**: Цвет кнопки. Возможные значения: `default` (серый), `primary` (фиолетовый/акцентный), `success` (зеленый), `warning` (оранжевый), `danger` (красный). Разрешается не указывать (по умолчанию `default`).
- **`style`**: Стиль кнопки. Возможные значения: `filled` (сплошная заливка), `outline` (только контур), `ghost` (без фона, прозрачная), `glass` (полупрозрачный матовый "эффект стекла", загорающийся при нажатии).

> [!TIP] **Premium дизайн: стиль Glass**
> Если вы хотите сделать свой дашборд футуристичным, используйте `style: glass`. В сочетании с параметром `color` (например, `color: primary`), кнопка станет матовой, будет пропускать задний фон заметки через стекло, а её светящиеся элементы и тени будут меняться в зависимости от выбранного цвета.
> ```yaml
> ```om-button
> label: Стеклянная светящаяся кнопка
> icon: zap
> action: run-command
> command: "app:open-settings"
> style: glass
> color: primary
> ```
> ```

## ⚙️ Доступные действия (`action`)

Каждая кнопка должна иметь параметр `action`, который говорит плагину, что именно нужно сделать по клику. Ниже описаны все доступные действия и их параметры.

### 1. `create-note` — Создать новую заметку
Создает новую заметку. Может сразу вставить в нее текст из параметров или из файла-шаблона.
- **`filename`** (обязательно): Имя будущего файла.
- **`folder`**: В какой папке создать (по умолчанию — в корне).
- **`content`**: Текст, который будет написан внутри.
- **`template`**: Путь к заметке-шаблону, текст из которой скопируется в новую заметку.
- **`open`**: `true` или `false` (по умолчанию `true` — сразу открыть созданную заметку).

**Пример:**
```yaml
```om-button
label: Создать Дневник
icon: book
action: create-note
folder: Дневники
filename: "Дневник - {{date}}"
content: "# Мой день\n\n- "
color: success
```
```

### 2. `open-note` — Открыть существующую заметку
Быстрый переход к другой заметке (удобно для создания "дашбордов" и меню).
- **`path`** (обязательно): Путь к заметке (с или без `.md`).
- **`newTab`**: Открыть в новой вкладке (`true` / `false`).

**Пример:**
```yaml
```om-button
label: Открыть To-Do
icon: list-todo
action: open-note
path: "Задачи/Inbox"
style: outline
```
```

### 3. `append-to-note` — Дописать текст в другую заметку
Полезно для быстрого захвата идей. Добавляет ваш текст в конец или начало выбранной заметки.
- **`path`** (обязательно): В какую заметку писать.
- **`content`** (обязательно): Какой текст вставить.
- **`position`**: `end` (в конец) или `start` (в начало). По умолчанию `end`.

**Пример:**
```yaml
```om-button
label: Записать быструю идею
icon: lightbulb
action: append-to-note
path: Идеи
content: "\n- [ ] Новая идея (добавлена в {{time}})"
color: warning
```
```

### 4. `insert-template` — Вставить шаблон туда, где стоит курсор
Вставляет содержимое указанного шаблона прямо в то место вашей текущей заметки, где вы работаете.
- **`template`** (обязательно): Путь к файлу-шаблону.
- **`position`**: `cursor` (на место курсора), `start` (в начало), `end` (в конец).

**Пример:**
```yaml
```om-button
label: Вставить блок задач
icon: blocks
action: insert-template
template: "Шаблоны/Блок_задач"
```
```

### 5. `run-command` — Выполнить команду Obsidian
Мгновенно запускает любую внутреннюю команду Obsidian (например, открыть настройки или графы). 
- **`command`** (обязательно): Системный ID команды. Например: `app:open-settings`, `workspace:split-vertical`, `editor:toggle-source`.

**Пример:**
```yaml
```om-button
label: Открыть настройки
icon: settings
action: run-command
command: "app:open-settings"
style: ghost
```
```

### 6. `open-url` — Переход по ссылке
Открывает любой веб-сайт в вашем браузере.
- **`url`** (обязательно): Адрес сайта (https://...).

**Пример:**
```yaml
```om-button
label: Искать в Google
icon: search
action: open-url
url: "https://google.com"
```
```

### 7. `ai-prompt` — Выполнить AI-запрос
Отправляет запрос нейросети, скармливая ей вашу текущую заметку в качестве контекста, и вставляет ее ответ в текст!
- **`prompt`** (обязательно): Команда для ИИ.
- **`position`**: Куда вставить ответ: `cursor`, `start`, `end`. (Если вы кликаете вне заметки, он покажет уведомление с ответом).

**Пример:**
```yaml
```om-button
label: Сделать Саммари
icon: brain
action: ai-prompt
prompt: "Прочитай текущую заметку и сделай из нее короткую выжимку (буллитами)."
position: end
color: primary
```
```

### 8. `load-context` — Выбрать и загрузить контекст
Показывает красивое всплывающее окно поиска со всеми файлами в указанной папке. При выборе файла его содержимое мгновенно отправляется в AI чат в качестве контекста, а боковая панель чата автоматически открывается!
- **`folder`** (необязательно): Папка, в которой нужно искать файлы. Если не указана, будет предлагать все файлы в хранилище.

**Пример:**
```yaml
```om-button
label: Выбрать саммари
icon: library
action: load-context
folder: "🧠 Contexts"
color: primary
style: glass
```
```

## 🔮 Переменные (Магия автоматизации)

Вы можете использовать умные подстановки (переменные) в значениях таких полей как `filename`, `content`, `folder` и `url`.

### Стандартные переменные
- `{{date}}` — Текущая дата (например, 2026-03-09)
- `{{time}}` — Текущее время в формате, безопасном для имен файлов (например, 14-30)
- `{{datetime}}` — Дата и время (2026-03-09 14-30)
- `{{title}}` — Название заметки, в которой сейчас нажата кнопка
- `{{vault}}` — Название вашего хранилища (Vault)

### 🧠 Продвинутые переменные (Связи и AI)

**1. `{{note:Путь/к/Файлу}}`**
Позволяет кнопке прочитать и извлечь весь текст из другой заметки прямо в момент нажатия!
*Сценарий:* Вставить текст из файла "Задачи" в prompt для AI.
`content: "Вот мои дела: {{note:Задачи}}"`

**2. `{{ai:ваш запрос}}`**
Использует нейросеть для генерации кусочка текста прямо внутри параметра. 
*Сценарий:* Заставить ИИ придумать креативное название файла при его создании.
`filename: "Идея - {{ai:Придумай 3 случайных английских слова}}"`
