## Фича 1: AI Чат-ассистент (Groq) ✅ ГОТОВО

### Описание
Встроенный чат-ассистент в боковой панели Obsidian. Пользователь пишет сообщение — AI отвечает.

### Пользовательские сценарии
1. Пользователь нажимает иконку 💬 в ribbon → открывается панель чата справа
2. Пользователь вводит текст → нажимает Enter или кнопку отправки
3. Появляется индикатор набора (typing dots) → приходит ответ AI
4. Пользователь может очистить историю кнопкой "Clear"
5. Ошибки (нет API ключа, сеть) показываются как сообщение об ошибке в чате

### Настройки
- API ключ Groq (обязателен, без него AI не работает)
- Модель (выбор из списка)
- Temperature (0–1, ползунок)
- Max tokens (число)
- System prompt (текстовое поле)

### Технические детали
- API: Groq REST API через `requestUrl` (встроенный в Obsidian)
- UI: Preact-компонент `ChatApp.tsx` в `ItemView` боковой панели
- Хранение истории: в памяти (на время сессии), НЕ персистентно

## Фича 2: Кнопки-скрипты 🔜

### Описание
Пользователь вставляет в заметку специальный code block, который плагин рендерит как кнопку.
При нажатии кнопки выполняется предустановленное действие.

### Формат кнопки в заметке

Кнопка определяется через code block с языком `om-button` и YAML-содержимым:

```
 ```om-button
 label: Текст на кнопке
 icon: lucide-icon-name
 action: тип-действия
 param1: значение1
 param2: значение2
 ```
```

### Обязательные поля
| Поле | Тип | Описание |
|------|-----|----------|
| `label` | string | Текст, отображаемый на кнопке |
| `action` | string | Тип действия (см. таблицу ниже) |

### Опциональные поля
| Поле | Тип | Описание | По умолчанию |
|------|-----|----------|-------------|
| `icon` | string | Имя Lucide-иконки | нет иконки |
| `color` | string | Цвет кнопки: `default`, `primary`, `success`, `warning`, `danger` | `default` |
| `style` | string | Стиль: `filled`, `outline`, `ghost` | `filled` |

### Доступные действия

#### `create-note` — Создать новую заметку
| Параметр | Обязателен | Описание |
|----------|-----------|----------|
| `filename` | ✅ | Имя файла (без `.md`) |
| `folder` | ❌ | Путь к папке (по умолчанию — корень Vault) |
| `content` | ❌ | Содержимое заметки |
| `template` | ❌ | Путь к файлу-шаблону (содержимое копируется) |
| `open` | ❌ | Открыть после создания (`true`/`false`, по умолчанию `true`) |

**Поведение:**
- Если заметка уже существует → показать Notice "Заметка уже существует"
- Если указан `template` и `content` одновременно → `template` приоритетнее
- Если папка не существует → создать папку автоматически
- Переменные в `filename` и `content` подставляются (см. раздел Переменные)

#### `open-note` — Открыть заметку
| Параметр | Обязателен | Описание |
|----------|-----------|----------|
| `path` | ✅ | Путь к заметке (с или без `.md`) |
| `newTab` | ❌ | Открыть в новой вкладке (`true`/`false`, по умолчанию `false`) |

**Поведение:**
- Если заметка не найдена → показать Notice "Заметка не найдена"

#### `append-to-note` — Дописать текст в заметку
| Параметр | Обязателен | Описание |
|----------|-----------|----------|
| `path` | ✅ | Путь к заметке |
| `content` | ✅ | Текст для добавления |
| `position` | ❌ | `end` (по умолчанию) или `start` |

**Поведение:**
- Если заметка не найдена → создать её с указанным content
- Добавляет перевод строки перед вставкой

#### `insert-template` — Вставить шаблон в текущую заметку
| Параметр | Обязателен | Описание |
|----------|-----------|----------|
| `template` | ✅ | Путь к файлу-шаблону |
| `position` | ❌ | `cursor` (по умолчанию), `end`, `start` |

**Поведение:**
- Вставляет содержимое файла-шаблона в текущую активную заметку
- Если нет активного редактора → показать Notice "Нет активной заметки"

#### `run-command` — Выполнить команду Obsidian
| Параметр | Обязателен | Описание |
|----------|-----------|----------|
| `command` | ✅ | ID команды Obsidian (например `app:open-settings`) |

**Поведение:**
- Вызывает `app.commands.executeCommandById(command)`
- Если команда не найдена → показать Notice "Команда не найдена: ..."

#### `open-url` — Открыть URL
| Параметр | Обязателен | Описание |
|----------|-----------|----------|
| `url` | ✅ | URL для открытия |

**Поведение:**
- Открывает URL во внешнем браузере через `window.open(url)`

### Переменные в параметрах

В значениях `filename`, `content`, `folder` можно использовать переменные:

| Переменная | Подстановка | Пример |
|-----------|-------------|--------|
| `{{date}}` | Текущая дата (YYYY-MM-DD) | `2026-03-07` |
| `{{time}}` | Текущее время (HH:mm) | `16:45` |
| `{{datetime}}` | Дата и время | `2026-03-07 16:45` |
| `{{title}}` | Название текущей заметки | `Мой проект` |
| `{{vault}}` | Имя текущего Vault | `MyVault` |

### Пользовательские сценарии

**Сценарий 1: Быстрое создание дневника**
```
 ```om-button
 label: 📝 Новый дневник
 action: create-note
 folder: Дневник
 filename: {{date}} — Дневник
 template: Templates/Дневник
 color: primary
 ```
```

**Сценарий 2: Ссылка на часто используемую заметку**
```
 ```om-button
 label: 📋 Открыть ToDo
 action: open-note
 path: Tasks/ToDo
 icon: check-square
 style: outline
 ```
```

**Сценарий 3: Быстрое добавление идеи**
```
 ```om-button
 label: 💡 Записать идею
 action: append-to-note
 path: Ideas/Inbox
 content: "\n- [ ] {{datetime}}: "
 color: success
 ```
```

### Edge Cases и ошибки
- Невалидный YAML → показать сообщение об ошибке вместо кнопки
- Неизвестный `action` → показать кнопку как неактивную с tooltip "Неизвестное действие"
- Пустой `label` → использовать название action как fallback
- Кнопка в режиме чтения (preview) → рендерится и работает
- Кнопка в режиме редактирования (source) → показывается как обычный code block

## Фича 3: AI + Работа с заметками (Фаза 4)

### Описание
AI ассистент получает доступ к Vault. Может читать, создавать и редактировать заметки.
Действия выполняются через чат или через кнопки-скрипты (action: `ai-prompt`).

### Действия AI с Vault

| Действие | Как пользователь просит | Что делает AI |
|----------|------------------------|---------------|
| Прочитать заметку | "Прочитай заметку Проект X" | Загружает контент → включает в контекст → отвечает |
| Создать заметку | "Создай заметку Идеи с планом" | Генерирует текст → создаёт файл в Vault |
| Редактировать | "Добавь в заметку Проект X список задач" | Читает → модифицирует → сохраняет |
| Поиск | "Найди заметки с тегом #работа" | Ищет по Vault → возвращает список |
| Суммаризация | "Перескажи заметку Встреча" | Читает → генерирует краткое содержание |

### Механика работы
1. AI получает расширенный system prompt с описанием доступных инструментов
2. Когда AI хочет выполнить действие — он возвращает структурированную команду
3. Плагин парсит ответ, извлекает команду, выполняет через `VaultService`
4. Результат операции возвращается AI как контекст для следующего ответа

### Контекст текущей заметки
- Если пользователь открыл заметку и пишет в чат — AI видит название и содержимое текущей заметки
- Это включается автоматически (без явного запроса пользователя)

### Кнопка с AI-действием
```
 ```om-button
 label: 🤖 Суммаризировать
 action: ai-prompt
 prompt: "Сделай краткое содержание текущей заметки"
 ```
```

### Ограничения и безопасность
- AI НЕ может удалять заметки (только создавать и редактировать)
- AI НЕ может выполнять произвольные команды Obsidian
- Все действия AI логируются в чате (пользователь видит что произошло)
- Перед созданием/редактированием AI показывает превью и спрашивает подтверждение

## Общие правила UX

### Обратная связь
- Успешные действия → `Notice` (зелёная полоска) на 3 секунды
- Ошибки → `Notice` с описанием проблемы на 5 секунд
- Длительные операции → индикатор загрузки

### Язык интерфейса
- Внутренний код и ID → английский
- UI-тексты и Notice → английский (пользователь может видеть разные локали Obsidian)
- AI отвечает на языке пользователя (указано в system prompt)
