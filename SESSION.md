## 🗺 План разработки (обновлённый)

### Фаза 3 — Кнопки-скрипты 🔜 ← ТЕКУЩАЯ
> Пользователь вставляет кнопки прямо в заметки. Каждая кнопка выполняет предустановленное действие.

#### Как работает:
Пользователь пишет в заметке code block:

````
```om-button
label: Создать дневник
icon: file-plus
action: create-note
template: Шаблон дневника
folder: Дневник
filename: {{date}} — Дневник
```
````

Плагин рендерит это как красивую кнопку, при нажатии выполняется действие.

#### Предустановленные действия:
| Action | Описание | Параметры |
|--------|----------|-----------|
| `create-note` | Создать новую заметку | `folder`, `filename`, `template`, `content` |
| `open-note` | Открыть существующую заметку | `path` |
| `insert-template` | Вставить шаблон в текущую заметку | `template` |
| `run-command` | Выполнить команду Obsidian | `command` (ID команды) |
| `open-url` | Открыть URL в браузере | `url` |
| `append-to-note` | Дописать текст в заметку | `path`, `content` |
| `toggle-checkbox` | Переключить чекбокс | — |

## 💡 Принятые архитектурные решения

- **UI фреймворк:** Preact (~3KB), монтируется в ItemView и Modal через `render()`
- **Стили:** CSS-переменные Obsidian, префикс классов `om-`
- **Сеть:** Только `requestUrl` (встроенный в Obsidian)
- **Модульность:** Каждая фича в отдельной папке → класс, реализующий `PluginModule`
- **Код:** TypeScript strict, main.ts минимальный
- **Модули:** `ModuleRegistry` загружает/выгружает в порядке регистрации
- **Кнопки:** Code block `om-button` с YAML-конфигурацией действий
- **AI + Vault:** AI получает доступ к заметкам через `VaultService`
- **Vault деплой:** `C:\Synchronization\.obsidian\plugins\obsidian-maker\`
